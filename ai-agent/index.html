üî¨ Research Findings: MCP & LSP

Model Context Protocol (MCP)

What is MCP?

The Model Context Protocol (MCP) is an open standard introduced by Anthropic in November 2024 to standardize how AI systems integrate with external tools and data sources. Think of it as "USB-C for AI applications."

  • Industry Adoption (2025): OpenAI integrated MCP in March 2025, Google DeepMind confirmed support in April 2025
  • Ecosystem: Thousands of MCP servers built, SDKs for all major languages
  • Tool Calling: Standardized approach for AI agents to execute tools and access data
  • Code Execution: Enables agents to load tools on demand and execute complex logic efficiently

Language Server Protocol (LSP) Integration

LSP for AI Agents

LSP integration allows AI agents to access code intelligence features like hover documentation, diagnostics, and code completion.

  • Agent Client Protocol (ACP): Emerging as "LSP for AI agents" (launched August 2025 with Zed Editor)
  • LSP-MCP Bridge: Allows AI assistants to perform advanced code analysis
  • LSP-AI: Open-source language server providing AI-powered code assistance

üöÄ Getting Started

Prerequisites

  • Python 3.9+ (for Python implementations)
  • Go 1.21+ (for Go implementation)
  • API keys for chosen providers (OpenAI, Anthropic, or Ollama running locally)
  • Obsidian vault (optional, but recommended)

Environment Variables

# Required for cloud providers
export OPENAI_API_KEY="your-openai-key"
export ANTHROPIC_API_KEY="your-anthropic-key"

# Optional: Specify vault location
export OBSIDIAN_VAULT_PATH="/path/to/vault"

# For Ollama (if not using default)
export OLLAMA_BASE_URL="http://localhost:11434"

Choosing an Implementation

Decision Guide:
  • Python TUI: Best for personal use, quick testing, and command-line workflows
  • Python Web: Best for team access, web-based workflows, and multiple users
  • Go TUI: Best for performance-critical applications and minimal dependencies
  • MCP Server: Best for standard integrations (Claude Desktop, etc.)

Adding Custom Tools

All implementations support adding custom tools. Here's how:

Python Example:

from agent import Tool

# Define your tool
my_tool = Tool(
    name="my_custom_tool",
    description="Does something useful",
    parameters={
        "type": "object",
        "properties": {
            "param1": {
                "type": "string",
                "description": "First parameter"
            }
        },
        "required": ["param1"]
    },
    function=lambda param1: f"Result: {param1}"
)

# Register it
tool_registry.register(my_tool)

Go Example:

registry.Register(Tool{
    Name: "my_custom_tool",
    Description: "Does something useful",
    Parameters: map[string]interface{}{
        "type": "object",
        "properties": map[string]interface{}{
            "param1": map[string]interface{}{
                "type": "string",
                "description": "First parameter",
            },
        },
        "required": []string{"param1"},
    },
    Function: func(args map[string]interface{}) (interface{}, error) {
        param1 := args["param1"].(string)
        return fmt.Sprintf("Result: %s", param1), nil
    },
})

üèóÔ∏è Architecture

Common Architecture Pattern

User Interface Layer
TUI (Textual/Bubble Tea) | Web (FastAPI + HTML/JS) | MCP Protocol
Agent Core Layer
Message Management | Provider Abstraction | Tool Registry
Provider Layer
OpenAI Provider | Anthropic Provider | Ollama Provider
Tool Layer
Obsidian Tools | Custom Tools | Future Extensions

Key Design Principles

  • Provider Abstraction: Unified interface for all LLM providers
  • Tool Registry Pattern: Pluggable tool system for easy extension
  • Error Handling: Comprehensive error handling at all layers
  • Type Safety: Strong typing in Go, Pydantic models in Python
  • Separation of Concerns: Clear separation between UI, logic, and tools

Extension Points

How to Extend:
  • Add New Providers: Implement the Provider interface
  • Add New Tools: Register tools via the Tool Registry
  • Custom UIs: Use the core agent logic with your own interface
  • Data Sources: Create tool wrappers for any data source
